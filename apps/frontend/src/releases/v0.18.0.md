# Version v0.18.0 Queen's Court

**Released:** 2026-03-28

## 🚀 What's New

### Queen History Tracking

Full queen lifecycle management has arrived! Beekeepers can now track individual queens across hives, record transfers with dates and reasons, and view the complete history of queens in any hive.

- **Queen List Page** (`/queens`) — browse all queens with filtering by status (Active, Replaced, Dead, Unknown) and hive assignment
- **Queen Detail Page** — view detailed queen information including color, marking, year, source, and full movement history between hives
- **Queen Transfer** — record queen transfers between hives with date, reason, and notes
- **Queen History Tab** — new tab on the hive detail page showing all queens that have been in the hive, with action buttons to add or replace queens

### Improved Queen Management in Hive Detail

The queen section in the hive detail header now provides better visibility and quick actions:

- Queen marking is clickable and links directly to the queen detail page
- "View Details" option added to the queen actions dropdown menu
- When no queen is present, a clear "No active queen" message is shown with an "Add Queen" button
- Mark as Dead and Mark as Lost/Missing actions now work — they update the queen status via the API and set the replaced date

### Queen History Tab Actions

The Queen History tab on the hive detail page now includes action buttons at the top:

- **View Details** — navigate to the current queen's detail page (when a queen is active)
- **Add Queen / Replace Queen** — contextual button that shows "Add Queen" when no queen is present, or "Replace Queen" when one is active
- Empty state also includes an "Add Queen" button for quick access

## 🐛 Bug Fixes

- Fixed `getColor` not defined error in the queen information component — replaced with the correct `getQueenColorClass` function
- Fixed "Mark as Dead" and "Mark as Lost/Missing" actions which were previously no-ops (only logged to console)
- Deduplicated queen actions dropdown in mobile and inline views — now reuses a shared menu component
- Fixed hive status management issues

## 🔧 Technical Improvements

- New `QueenMovement` database model for tracking queen transfers between hives
- New API endpoints for queen history (`/api/queens/hive/:hiveId/history`), queen movement history (`/api/queens/:id/history`), and queen transfer (`/api/queens/:id/transfer`)
- `QueenColorBadge` extracted as a reusable component
- Shared queen constants extracted to reduce code duplication
- New React Query hooks: `useQueenHistory`, `useHiveQueenHistory`, `useRecordQueenTransfer`
- Translation keys added for all new queen actions across all supported languages (English, German, French, Italian, Slovak, Danish, Serbian)
